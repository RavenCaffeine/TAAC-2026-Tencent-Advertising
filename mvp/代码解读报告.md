# KDD 2026 竞赛 Baseline 代码详细解读报告

## 一、竞赛概述

### 1.1 赛题背景

本次竞赛主题为**"面向大规模推荐的序列建模与特征交互的统一"**，由腾讯广告发起。核心目标是在单一架构内联合建模**序列用户行为**和**非序列多字段特征**，用于 **CVR（转化率）预测**。

### 1.2 评估指标

- **主指标**：AUC（ROC曲线下面积），越高越好
- **约束条件**：推理延迟必须在规定预算内，超时提交无效
- **当前 Baseline 成绩**：验证集 AUC = 0.8615，评测集 AUC = 0.8106，推理时间 371.82s

### 1.3 数据集规模

- **2亿条**用户序列，扁平列布局
- **120列**特征，分6大类：ID和标签(5)、用户离散特征(46)、用户稠密特征(10)、物品离散特征(14)、域序列特征(45)
- 4个行为域序列：domain_a(9列)、domain_b(14列)、domain_c(12列)、domain_d(10列)

## 三、数据管道详解 (`dataset.py`)

### 3.1 FeatureSchema 类

`FeatureSchema` 是特征元数据容器，记录每个特征的 `(feature_id, offset, length)` 三元组。它为下游代码提供按 `fid` 定位扁平张量中对应位置的能力。

- **单值特征**：`length = 1`，如用户年龄
- **多值特征**：`length = array_length`，如用户兴趣标签列表

### 3.2 时间桶编码 (Time Bucketing)

系统定义了 64 个时间差边界（从5秒到1年），将序列中每个事件与当前时间戳的差值映射到 65 个桶（0=padding, 1~64 为有效桶）。这是一种**相对时间位置编码**方案：

```python
BUCKET_BOUNDARIES = [5, 10, 15, ..., 86400, 172800, ..., 31536000]
NUM_TIME_BUCKETS = 65  # len(BUCKET_BOUNDARIES) + 1
```

时间差越小桶越密集（秒级精度），越大越稀疏（从天到年），体现了"近期行为更重要"的直觉。

### 3.3 PCVRParquetDataset 类

这是核心数据集类，继承自 `IterableDataset`，具备以下优化：

**性能优化手段：**
1. **预分配 NumPy Buffer**：避免每个 batch 反复分配内存（`_buf_user_int`, `_buf_seq` 等）
2. **预计算列索引查找**：`_col_idx` 字典避免每行都做字符串查找
3. **融合填充循环**：序列特征直接写入预分配的 3D Buffer，一次遍历完成
4. **`file_system` 共享策略**：避免 `/dev/shm` 在多 Worker 场景下耗尽

**数据处理流程：**
1. 读取 Parquet Row Group → Arrow RecordBatch
2. 用户离散特征 → 填充到 `(B, total_dim)` 张量，负值/null 映射为 0
3. 用户稠密特征 → 变长浮点数组 pad 到固定长度
4. 序列特征 → 按域分组，pad 到 `(B, n_feats, max_len)` 3D 张量
5. 时间戳列 → 计算时间差 → `np.searchsorted` 桶化
6. **Out-of-bound 处理**：超出 vocab 范围的 ID clip 到 0，并记录统计

**Shuffle 策略：** 以 `buffer_batches` 为单位缓冲多个 batch，行级随机打乱后重新切片。这是对 `IterableDataset` 无法全局 shuffle 的实用折衷。

### 3.4 数据切分

```python
# 按 Row Group 切分：前 N% 训练，后 valid_ratio% 验证
n_valid_rgs = max(1, int(total_rgs * valid_ratio))  # 默认 10%
n_train_rgs = total_rgs - n_valid_rgs
```

## 四、模型架构详解 (`model.py`)

### 4.1 整体架构：PCVRHyFormer

PCVRHyFormer 是一个**混合 Transformer 模型**，核心思想来自 HyFormer 论文，在单一架构中统一处理序列行为和非序列多字段特征。

**前向传播流程：**

```
输入特征
  ├── 用户离散特征 ──→ NS Tokenizer ──→ User NS tokens (B, num_user_groups, D)
  ├── 用户稠密特征 ──→ Linear+LN    ──→ User Dense token (B, 1, D)
  ├── 物品离散特征 ──→ NS Tokenizer ──→ Item NS tokens (B, num_item_groups, D)
  ├── 物品稠密特征 ──→ Linear+LN    ──→ Item Dense token (B, 1, D)
  │
  │                     拼接 ──→ NS Tokens (B, num_ns, D)
  │
  ├── 序列域A ──→ Embedding+Proj ──→ Seq tokens A (B, L_a, D)
  ├── 序列域B ──→ Embedding+Proj ──→ Seq tokens B (B, L_b, D)
  ├── 序列域C ──→ Embedding+Proj ──→ Seq tokens C (B, L_c, D)
  └── 序列域D ──→ Embedding+Proj ──→ Seq tokens D (B, L_d, D)
        │
        ▼
  MultiSeqQueryGenerator ──→ Q tokens per seq (B, Nq, D) × 4
        │
        ▼
  ┌─ MultiSeqHyFormerBlock ×N ──────────────────────────┐
  │  1. Sequence Evolution：各序列独立编码（Self-Attn）    │
  │  2. Query Decoding：Q tokens Cross-Attn 到序列       │
  │  3. Token Fusion：拼接所有 Q + NS tokens             │
  │  4. Query Boosting：RankMixer（Token Mixing + FFN）   │
  └──────────────────────────────────────────────────────┘
        │
        ▼
  Output Projection → Classifier → logits (B, 1)
```

### 4.2 NS Tokenizer（非序列特征编码）

提供两种变体：

**GroupNSTokenizer（`ns_tokenizer_type='group'`）：**
- 将离散特征按语义分组（如 ns_groups.json 定义的 U1~U7, I1~I4）
- 每组特征分别 Embedding → Concat → Linear+LN → SiLU → 1个 NS token
- 多值特征通过 Mean Pooling 聚合（忽略 padding=0）
- 输出 token 数 = 分组数

**RankMixerNSTokenizer（`ns_tokenizer_type='rankmixer'`，默认）：**
- 所有特征 Embedding 后 Concat 成一个长向量
- 等分为 `num_ns_tokens` 段，每段独立投影到 `d_model`
- token 数可自由设置，与分组数解耦
- 当前配置：user_ns_tokens=5, item_ns_tokens=2

### 4.3 序列特征编码

**Embedding 层：**
- 每个序列域的每个子特征（side-info）有独立 Embedding
- 高基数特征（vocab > `seq_id_threshold=10000`）训练时加倍 Dropout（`dropout_rate * 2`）
- 超高基数特征（vocab > `emb_skip_threshold=1000000`）跳过 Embedding，输出零向量
- 所有子特征 Embedding Concat → Linear+LN+GELU → `(B, L, D)`
- 可选加上时间桶 Embedding

**序列编码器（3种选择）：**

| 编码器 | 结构 | 复杂度 | 适用场景 |
|--------|------|--------|----------|
| `swiglu` | LN → SwiGLU → Dropout + Residual | O(L) | 速度优先，无注意力 |
| `transformer`（默认） | Pre-LN Self-Attn + FFN + Residual | O(L²) | 标准 Transformer |
| `longer` | Top-K 压缩 + Cross/Self-Attn | O(L·K) | 长序列压缩 |

### 4.4 MultiSeqQueryGenerator（查询生成器）

为每个序列域独立生成 `num_queries` 个 Q token：

```
GlobalInfo_i = Concat(NS_tokens.flatten(), MeanPool(Seq_i))
Q_{i,j} = FFN_{i,j}(LayerNorm(GlobalInfo_i))    j = 1..Nq
```

关键设计：每个序列的全局信息包含所有 NS token + 该序列的均值池化表示，使 Q token 同时感知用户/物品属性和序列语义。

### 4.5 MultiSeqHyFormerBlock（核心堆叠块）

每个块执行 4 步操作：

1. **Sequence Evolution**：各序列独立自编码（TransformerEncoder/SwiGLU/Longer）
2. **Query Decoding**：Q tokens 通过 Cross-Attention 从进化后的序列中提取信息
3. **Token Fusion**：将所有序列的 Q tokens 和 NS tokens 拼接
4. **Query Boosting（RankMixer）**：
   - `full` 模式：Token Mixing（无参数 reshape 交换子空间）+ Shared FFN + Residual + Post-LN
   - `ffn_only` 模式：仅 FFN，跳过 Token Mixing
   - `none` 模式：恒等映射

Token Mixing 的核心是一个无参数的 reshape 操作：将 `(B, T, D)` reshape 为 `(B, T, T, d_sub)` 再 transpose，实现跨 token 的信息交换。约束：`d_model` 必须被 `T` 整除。

### 4.6 RoPE（旋转位置编码）

可选特性（`--use_rope`），默认关闭。预计算 cos/sin 缓存，在 Self-Attn 中注入位置信息。LongerEncoder 的 Cross-Attn 通过 `gather` 从全局位置表中按 top-k 位置索引取 Q 侧 RoPE。

### 4.7 门控注意力

`RoPEMultiheadAttention` 在输出投影前加入了**门控机制**：
```python
G = W_g(query)          # 门控向量
out = out * sigmoid(G)  # 门控注意力输出
out = W_o(out)          # 最终投影
```
`W_g` 初始化为 bias=1, weight=0，即初始时门完全打开，训练过程中逐渐学习抑制不重要信息。

### 4.8 分类头

```python
self.clsfier = nn.Sequential(
    nn.Linear(d_model, d_model),
    nn.LayerNorm(d_model),
    nn.SiLU(),
    nn.Dropout(dropout_rate),
    nn.Linear(d_model, action_num)  # action_num=1
)
```

## 五、训练流程详解 (`trainer.py`)

### 5.1 双优化器策略

```
Embedding 参数 → Adagrad（lr=0.05）  ← 适合稀疏更新
其他参数       → AdamW（lr=1e-4, betas=(0.9, 0.98)）
```

这是推荐系统的经典做法：Embedding 表更新稀疏且幅度大，Adagrad 自适应学习率适合；Dense 参数用 AdamW 更稳定。

### 5.2 损失函数

支持两种：
- **BCE（默认）**：`F.binary_cross_entropy_with_logits`
- **Focal Loss**：`alpha=0.1, gamma=2.0`，关注难样本，缓解类别不平衡

### 5.3 高基数 Embedding 冷重启

每个 epoch 结束后（从第 `reinit_sparse_after_epoch` 个 epoch 起），重新初始化 vocab > `reinit_cardinality_threshold` 的 Embedding 表，并重建 Adagrad 优化器状态。低基数 Embedding 的优化器状态被保留。

这一技巧来自快手团队的 MultiEpoch 论文，目的是防止高基数特征过拟合——因为这些特征的 ID 在多轮训练中被反复看到，容易记住训练集噪声。

### 5.4 训练循环

```
for epoch in [1, num_epochs]:
    for step, batch in train_loader:
        1. batch_to_device（non_blocking + pin_memory）
        2. forward → logits
        3. compute loss（BCE / Focal）
        4. backward + clip_grad_norm(max_norm=1.0)
        5. dense_optimizer.step + sparse_optimizer.step
        6. 每 eval_every_n_steps 步执行验证
    
    epoch 末执行验证
    EarlyStopping 检查（patience=5）
    高基数 Embedding 冷重启
```

### 5.5 Checkpoint 管理

- 每次 best_model 更新时保存 `global_stepN.best_model/` 目录
- 包含 `model.pt`、`schema.json`、`ns_groups.json`、`train_config.json`
- 旧的 best_model 目录自动清理
- 设计为自包含：推理时只需 checkpoint 目录即可完整重建模型

## 六、推理流程详解 (`infer.py`)

### 6.1 流程概要

```
1. 从环境变量读取：MODEL_OUTPUT_PATH, EVAL_DATA_PATH, EVAL_RESULT_PATH
2. 加载 train_config.json → 解析全部模型超参数
3. 加载 schema.json → 构建 PCVRParquetDataset
4. 构建 PCVRHyFormer → 严格加载 state_dict
5. 遍历测试数据 → model.predict() → sigmoid → 概率
6. 输出 predictions.json：{user_id: probability}
```

### 6.2 关键设计

- **超参数回溯**：所有模型结构参数从 `train_config.json` 恢复，缺失时回退到 `_FALLBACK_MODEL_CFG`
- **Strict Load**：`model.load_state_dict(strict=True)` 确保模型结构与权重完全匹配
- **predict vs forward**：`predict()` 关闭 Dropout，返回 logits + embeddings

## 八、代码亮点与不足分析

### 8.1 亮点

1. **统一架构设计**：通过 NS Tokenizer + MultiSeqQueryGenerator + HyFormerBlock，将异质特征统一到 token 空间
2. **双优化器策略**：Adagrad(Sparse) + AdamW(Dense) 是推荐系统最佳实践
3. **高基数冷重启**：MultiEpoch 技巧有效防止 Embedding 过拟合
4. **数据管道优化**：预分配 Buffer、融合填充、file_system 共享策略
5. **时间桶编码**：非均匀桶化方案捕捉时间衰减效应
6. **门控注意力**：W_g 门控提升注意力选择性
7. **自包含 Checkpoint**：训练配置完整保存，推理可独立运行

### 8.2 不足与改进方向

1. **模型容量偏小**：d_model=64, 2层 block，参数量有限
2. **缺乏学习率调度**：无 warmup、无 cosine decay
3. **无混合精度训练**：未使用 AMP，训练速度和显存利用均有提升空间
4. **序列编码器单一**：所有域共用同一类型编码器，未针对域特性差异化
5. **无 Target-Aware Attention**：未利用当前候选物品信息引导序列注意力
6. **Dense 特征利用不足**：仅一个 token，未与 Int 特征做交叉
7. **验证-评测 Gap 较大**：过拟合问题需解决（更强正则化 / 更多数据）
8. **推理时间偏高**：371s，需优化以满足延迟限制
