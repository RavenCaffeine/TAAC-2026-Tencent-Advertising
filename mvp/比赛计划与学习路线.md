# KDD 2026 竞赛计划与学习路线

## 一、竞赛目标分析

### 1.1 赛道目标

| 目标 | 说明 | 评判标准 |
|------|------|----------|
| 排行榜排名 | AUC 越高越好 | 自动评测 AUC |
| 统一模块创新奖 ($45,000) | 统一序列建模与特征交互的架构创新 | 论文评审，重创新性 |
| 扩展规律创新奖 ($45,000) | 探索推荐系统的 Scaling Law | 论文评审，重洞见 |

- Baseline AUC：0.8106（评测集）/ 0.8615（验证集）
- 过拟合 Gap：~0.05（需优化正则化）
- 推理时间：371.82s（需关注延迟限制）

## 二、分阶段比赛计划

### Phase 1：基础夯实（第1-2周）

**目标**：理解代码 + 复现 baseline + 搭建实验环境

1. **完整阅读并理解 Baseline 代码**（参考代码解读报告）
2. **搭建实验环境**
   - 配置 GPU 服务器（建议 A100/H100 或 4×V100）
   - 安装依赖：PyTorch 2.x, PyArrow, sklearn, tensorboard
   - 验证 `run.sh` 能正常训练并复现 AUC ≈ 0.86
3. **搭建实验追踪系统**
   - 使用 Weights & Biases 或 TensorBoard 记录所有实验
   - 建立超参数搜索的标准流程
4. **阅读核心论文**
   - HyFormer: 重新审视序列建模和特征交互 [3]
   - InterFormer: 异构交互学习 [1]
   - OneTrans: 基于单 Transformer 的统一推荐 [2]
   - DIN / DIEN / SIM: 序列推荐经典论文
   - DCN v2 / AutoInt / FinalMLP: 特征交互经典论文

### Phase 2：快速涨分——Low-Hanging Fruit（第3-4周）

**目标**：通过工程优化和简单改进，将 AUC 提升 0.5-1%

**优先级从高到低：**

1. **混合精度训练（AMP）**
   - 加入 `torch.cuda.amp.autocast()` + `GradScaler`
   - 预期效果：训练速度提升 1.5-2x，可尝试更大 batch_size

2. **学习率调度器**
   - 加入 Warmup + Cosine Annealing
   - Warmup 步数建议：总步数的 5-10%

3. **增大模型容量**
   - `d_model`: 64 → 128 → 256
   - `num_hyformer_blocks`: 2 → 3 → 4
   - `num_heads`: 4 → 8
   - 注意 RankMixer `full` 模式的 T 整除约束

4. **Focal Loss 调参**
   - 当前 BCE 可能不是最优，尝试 Focal Loss
   - alpha 和 gamma 网格搜索

5. **序列长度调优**
   - 尝试增大 seq_max_lens（如 c:1024, d:1024）
   - 对比不同截断长度对 AUC 的影响

6. **Embedding 维度调优**
   - `emb_dim`: 64 → 32（可能减少过拟合）或 128（增加表达力）
   - 调整 `emb_skip_threshold` 和 `seq_id_threshold`

### Phase 3：架构创新——核心竞争力（第5-8周）

**目标**：设计统一架构创新，冲击 AUC + 创新奖

1. **Target-Aware Attention（重要！）**
   - 在 Cross-Attention 中引入目标物品信息
   - Q tokens 不仅来自全局信息，还应编码当前候选物品
   - 参考 DIN/DIEN 的 Target Attention 思路

2. **统一分词方案设计**
   - 当前 NS 特征和 Seq 特征用不同方式编码
   - 设计统一的 tokenizer：将所有特征（无论序列/非序列）映射到同一 token 空间
   - 这是比赛鼓励的核心创新方向

3. **同构可堆叠主干架构**
   - 设计一种块结构，对序列和非序列特征使用相同的处理方式
   - 参考 OneTrans 的统一 Transformer 方法
   - 确保块可以任意堆叠，形成 Scaling 友好的架构

4. **多域序列交互**
   - 当前 4 个域序列完全独立编码
   - 引入跨域注意力或共享参数
   - 域间信息互补可能提升效果

5. **Dense-Sparse 特征交叉**
   - 当前 dense 特征仅做简单投影
   - 引入 DCN v2 风格的显式交叉层
   - 或在 NS token 阶段融合 dense 和 int 特征

6. **LongerEncoder 优化**
   - 默认 transformer 编码器对长序列效率低
   - 切换到 `longer` 编码器并调优 `top_k`
   - 或引入 Flash Attention / 线性注意力

### Phase 4：Scaling Law 探索（第6-9周）

**目标**：系统性研究推荐系统的扩展规律，冲击创新奖

1. **模型规模扩展实验**
   - 固定数据量，变化模型参数：d_model ∈ {32, 64, 128, 256, 512}
   - 记录 AUC vs 参数量 的曲线
   - 观察是否存在类似 LLM 的 Power Law

2. **数据规模扩展实验**
   - 固定模型，变化训练数据量：train_ratio ∈ {0.1, 0.2, 0.5, 1.0}
   - 记录 AUC vs 数据量 的曲线

3. **序列长度扩展实验**
   - 变化 seq_max_lens：{64, 128, 256, 512, 1024}
   - 记录 AUC vs 序列长度

4. **Embedding 维度扩展实验**
   - 变化 emb_dim：{16, 32, 64, 128}
   - 分析 Embedding 参数量的贡献

5. **总结 Scaling Law**
   - 拟合幂律曲线，给出经验公式
   - 分析瓶颈在哪：数据量？模型深度？序列长度？

### Phase 5：集成与冲刺（第9-10周）

**目标**：最终提升排名

1. **模型集成**
   - 训练多个不同配置的模型
   - 简单平均 / 加权平均 / Stacking
   - 注意推理时间限制！

2. **推理优化**
   - TorchScript / torch.compile 加速
   - 量化推理（INT8）
   - 批处理优化
   - 目标：推理时间 < 200s

3. **后处理**
   - 概率校准（Platt Scaling / Isotonic Regression）
   - 分段优化（如按用户活跃度分段）

4. **论文撰写**
   - 总结架构创新点
   - 展示 Scaling Law 实验结果
   - 准备技术报告

## 三、学习路线

### 3.1 必读论文（按优先级排序）

**竞赛直接相关：**
1. HyFormer (2026) - Baseline 架构来源
2. InterFormer (CIKM 2025) - 异构特征交互
3. OneTrans (2025) - 统一 Transformer 推荐

**序列推荐基础：**
4. DIN (KDD 2018) - Deep Interest Network，目标注意力
5. DIEN (AAAI 2019) - 兴趣演化网络
6. SIM (CIKM 2020) - 长序列建模
7. BST (DLP-KDD 2019) - Behavior Sequence Transformer
8. SDIM (CIKM 2022) - 高效长序列建模

**特征交互基础：**
9. DCN v2 (WWW 2021) - 显式交叉网络
10. AutoInt (CIKM 2019) - 自注意力特征交互
11. FinalMLP (AAAI 2023) - 双流 MLP 交互
12. GDCN (WWW 2024) - 门控交叉网络

**位置编码与注意力：**
13. RoPE (2021) - 旋转位置编码
14. Flash Attention (NeurIPS 2022) - 高效注意力
15. ALiBi (ICLR 2022) - 线性偏置注意力

**Scaling Laws：**
16. Chinchilla (2022) - LLM Scaling Laws
17. Scaling Laws for Neural Language Models (2020)

### 3.2 技术栈学习

**必须掌握：**
- PyTorch 2.x（特别是 torch.compile, AMP, FSDP）
- PyArrow / Parquet 数据格式
- TensorBoard / W&B 实验追踪
- Linux 命令行 + Shell 脚本
- Git 版本管理

**建议掌握：**
- CUDA 基础（理解 GPU 内存模型）
- Mixed Precision Training 原理
- Distributed Data Parallel (DDP)
- ONNX Runtime / TensorRT（推理加速）

## 四、关键风险与应对

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| 推理超时 | 提交无效 | 优先控制模型复杂度，定期测推理时间 |
| 过拟合（Gap 0.05） | 排行榜下滑 | 增强正则：Dropout、Label Smoothing、数据增强 |
| GPU 资源不足 | 无法跑大模型 | AMP + 梯度累积 + DeepSpeed |
| 论文创新性不够 | 失去创新奖 | Phase 3-4 重点投入，有独到见解 |
| 代码 Bug | 浪费时间 | 单元测试 + 小规模快速验证 |
